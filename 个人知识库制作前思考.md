## 三、AI Agent 搭建与发布规划

### 1. 发布策略（基于 Coze）
- **阶段一：发布为网页**
  - 目的：直观看到可交互成果
  - 配套：更新技术博客，介绍网页功能与实现思路
- **阶段二：发布为 API**
  - 目的：用于个人二次开发（如接入现有 AI 大模型、集成到其他项目）

### 2. 技术栈选型
- **容器化部署**：Docker
  - 作用：打包 FastAPI 应用、MySQL/Redis 服务，实现环境隔离与一键部署
- **数据存储**：
  - MySQL：存储用户账号、权限等结构化数据
  - Redis：缓存对话记录、管理登录态、实现上下文记忆
- **后端开发**：Anaconda + Python + FastAPI
  - 封装 Coze 的 API 能力，构建自定义后端服务
  - 实现用户登录验证、对话记录持久化等扩展功能
  - 疑问：Coze 平台原生是否支持上下文记忆？是否需要通过 FastAPI 额外开发？

### 3. 代码与开源问题
- AI Agent 配置是否能直接上传到 GitHub？
- 若不能直接开源：在技术博客中写明 AI Agent 访问链接，说明实现思路与技术选型
